JavaScript Execution Context

Understanding the environment where JavaScript code is evaluated and executed

What is Execution Context?

An execution context is an abstract concept that represents the environment in which JavaScript code is evaluated and executed. Every time code runs in JavaScript, it runs inside an execution context.

Note: Understanding execution context is fundamental to mastering JavaScript, especially concepts like scope, closures, and the 'this' keyword.

Types of Execution Context

  • Global Execution Context: The default context where code that's not inside any function executes
  • Function Execution Context: Created whenever a function is invoked
  • Eval Execution Context: Code executed inside eval function (rarely used)

Components

  • Variable Environment: Stores variables and function declarations
  • Lexical Environment: Contains identifier-variable mapping
  • This Binding: Determines the value of 'this' keyword
