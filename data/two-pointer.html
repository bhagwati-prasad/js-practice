Encyclopedia of Two-Pointer Techniques

1. Converging Pointers (Opposite Ends)
Two pointers start at the extreme ends of a sorted structure and move inward.
Use Case Best for finding pairs or verifying global symmetry in sorted arrays or strings. Key Insight In a sorted array, moving the left pointer increases values and the right decreases them, allowing for a targeted linear search.
Common Applications
Two Sum Sorted Finds a pair that sums to a target by incrementing the left pointer for a larger sum or decrementing the right for a smaller one.
Valid Palindrome Simultaneously verifies characters from both ends of a string to ensure perfect symmetry in a single pass.
Container With Most Water Maximizes the area between two bars by moving the pointer pointing to the shorter height, seeking a taller boundary.
2. Slow-Fast Pointers (Tortoise and Hare)
Two pointers traverse a sequence at different speeds (usually 1x and 2x).
Use Case Used to identify structural properties like cycles or midpoints in linked lists where indexing is unavailable. Key Insight Relative motion in a cycle ensures collision; relative motion in a line ensures one pointer stays exactly at the halfway mark.
Common Applications
Cycle Detection Determines if a linked list contains a loop by checking if the fast pointer eventually overlaps the slow pointer.
Linked List Middle Locates the central node in one pass by moving the fast pointer two nodes for every one node moved by the slow pointer.
Happy Number Identifies if a chain of digit-square calculations reaches 1 or falls into an infinite loop by treating values as nodes in a graph.
3. Sliding Window (Fixed or Variable)
A contiguous range is maintained between pointers that slides across the data.
Use Case Optimal for finding sub-segments that satisfy specific mathematical or logical constraints. Key Insight Reduces O(N²) problems to O(N) by incrementally updating the window state rather than re-calculating from scratch.
Common Applications
Longest Unique Substring Uses a window to track distinct characters, contracting the left side whenever a duplicate is detected.
Minimum Window Substring Finds the smallest substring containing all characters of a pattern by expanding to find coverage and shrinking to find optimality.
Sliding Window Maximum Tracks the peak value within a moving frame of size K using a deque to maintain a list of potential maximum candidates.
4. Diverging Pointers (Expand from Center)
Pointers start at a central pivot and expand outward to the boundaries.
Use Case Used for identifying symmetry from the inside-out, such as finding palindromes. Key Insight Validating from the center outward allows for early exit as soon as symmetry is broken, optimizing symmetric checks.
Common Applications
Longest Palindromic Substring Examines every character and character-pair as a possible center, expanding until the substring is no longer symmetric.
Count Palindromic Substrings Iterates through all possible center indices and counts every valid expansion that remains a palindrome.
5. Multi-Structure Pointers (Independent)
Separate pointers are assigned to different data structures for simultaneous traversal.
Use Case Merging, intersecting, or comparing pre-sorted datasets in linear time. Key Insight Syncs multiple data streams by advancing pointers based on value comparisons across structures.
Common Applications
Merge Sorted Arrays Combines two sorted lists into one by comparing current pointers and selecting the smaller element at each step.
Is Subsequence Verifies if one string is a subsequence of another by matching characters from the first using the second as a search stream.
6. Read-Write Pointers (In-place)
A Read pointer scans the data while a Write pointer records the valid elements.
Use Case Highly space-efficient in-place modification and filtering of arrays. Key Insight Since the Read pointer is always >= Write pointer, we can safely overwrite the same array without losing future values.
Common Applications
Remove Duplicates Sorted Scans sorted data and 'writes' only unique values to the front of the array, effectively truncating duplicates.
Move Zeroes Collects all non-zero values at the front of the array and fills the remaining trailing space with zero values.
7. Saddleback Search (2D)
Navigates a 2D sorted matrix from a corner where directions offer binary trends.
Use Case Finding targets in large, multi-dimensionally sorted tables (like database indexes). Key Insight Every comparison allows the algorithm to discard an entire row or column, resulting in O(N+M) complexity.
Common Applications
Search 2D Matrix II Navigates from top-right to bottom-left to find a target by moving left for smaller values or down for larger ones.
Count Negatives in Sorted Matrix Identifies the 'boundary' between negative and positive numbers across the matrix to sum up total negative counts.
8. Binary Search (Logarithmic Pointers)
Pointers define a range that is halved by jumping to the mid-point.
Use Case Rapidly finding items in sorted arrays or optimizing a numeric result within a continuous range. Key Insight Halving the search space per step ensures logarithmic efficiency (O(log N)), which handles billions of records in seconds.
Common Applications
Rotated Array Search Identifies which half of a rotated array is sorted to correctly adjust boundaries for the target search.
Find Peak Element Navigates toward a local maximum by comparing the midpoint with its neighbors to find an 'uphill' slope.
9. Interval Pointers (Sweep Line)
Two pointers moving independently across sorted lists of start and end timestamps.
Use Case Solving scheduling, concurrency, and interval intersection problems. Key Insight Tracking events chronologically as 'entry' and 'exit' allows us to calculate simultaneous overlaps at any point in time.
Common Applications
Meeting Rooms II Determines the maximum number of simultaneous meetings by processing all start and end times in a single timeline.
Interval Intersection Finds all time segments where two different sets of intervals overlap by comparing endpoints in a two-pointer pass.
10. Triple Pointers (Partitioning)
Three pointers dividing an array into three logical categorical segments.
Use Case Sorting categorical data (like flags or colors) in a single linear pass. Key Insight Maintaining three distinct boundaries allows for O(N) sorting with constant extra memory.
Common Applications
Sort Colors (Dutch Flag) Organizes 0s, 1s, and 2s into their respective blocks by swapping values based on a scanning middle pointer.
QuickSort 3-way Partition Handles arrays with many duplicate keys by splitting elements into strictly less, strictly greater, and equal to the pivot.
11. Cyclic Pointers (Index as Pointer)
Treats the value found at an index as the 'next' pointer location.
Use Case Finding duplicate or missing numbers within an array of range-bound integers. Key Insight Maps the value-index relationship as a directed graph; logic like slow-fast pointers can then find cycles representing duplicates.
Common Applications
Duplicate Number Find Uses Floyd’s cycle detection on the array (index -> value) to find the entrance to a cycle, which identifies the duplicate.
First Missing Positive Rearranges the array so each number 'X' is placed at index 'X-1'; the first mismatch in a subsequent pass identifies the gap.
12. Two-Pass Greedy Pointers
Resolves bilateral dependencies by applying a uni-directional pass from both ends sequentially.
Use Case Used when an element's final value is influenced by its relationship with both left and right neighbors. Key Insight Decouples complex two-sided problems into two simple one-sided accumulation tasks.
Common Applications
Rain Water Trapping Calculates water levels by determining the highest bars to the left and right of each point in two linear passes.
Candy Distribution First pass satisfies left neighbor requirements; second pass corrects for right neighbors while maintaining the first pass's constraints.
